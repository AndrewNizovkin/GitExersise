* `git init` - инициализация локального репозитория
* `git status` - получить информацию от [git](https://ru.wikipedia.org/wiki/Git) о его текущем состояниии

* `git add` - добавить файл или файлы к следующему [коммиту](https://ru.wiktionary.org/wiki/%D0%BA%D0%BE%D0%BC%D0%BC%D0%B8%D1%82)

* `git commit -m "message"` - создание коммита.

* `git log` - вывод на экран истории всех коммитов с их [хэш-кодами](https://ru.wikipedia.org/wiki/%D0%A5%D0%B5%D1%88-%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D1%8F)

* `git checkout` - переход от одного коммита к другому

* `git checkout master` - вернуться к актуальному состоянию и продолжить работу

* `git checkout -f HEAD`  - переход на ветку (HEAD - текущая ветка) с сохранением. Флаг -f говорит, что нужно все файлы сохранить из той ветки на которую переключаемся. Для текущей ветки можно не указывать HEAD


* `git diff` - увидеть разницу между текущим файлом и закоммиченным файлом
